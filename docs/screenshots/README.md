# Скриншоты документации

Текущий набор, который используется в `README.md`:
- `home-feed.png` — главная страница/лента.
- `chat-page.png` — страница чатов.
- `radio.png` — раздел радио.
- `iptv-playing.png` — раздел IPTV во время воспроизведения.
- `cabinet.png` — личный кабинет пользователя; при обновлении скриншота желательно показывать форму нового поста с очередью загрузки и прогрессом.
- `admin-settings.png` — админ-панель (настройки).

Опционально для новой аналитики админки:
- `admin-dashboard.png` — вкладка дашборда (KPI, графики подписок и активности, retention/content/media/radio/IPTV analytics).
- `admin-dashboard-formulas-reference.png` — при необходимости отдельный скриншот блока аналитики, если обновляется `docs/analytics-metrics.md` и нужно показать заказчику сам экран, к которому относятся формулы.
- `cabinet-upload-progress.png` — отдельный акцентный скриншот очереди загрузки медиа в личном кабинете.
- `api-documentation.png` — Swagger/OpenAPI UI после `php artisan l5-swagger:generate`, если обновляется раздел API-документации в `README.md`.
- `home-back-to-top.png` — экран с глобальной кнопкой `В начало`, если обновляется описание навигации и адаптива.

Требования к новым скриншотам:
- Формат: `PNG`.
- Рекомендуемая ширина: `1200-1600 px`.
- Читабельный текст интерфейса без сильного масштабирования.
- Без личных данных в открытом виде.

Рекомендуется дополнительно держать актуальный mobile-снимок для радио (компактный блок "Сейчас играет", свернутые найденные станции, контрастные счетчики/чипы), если меняется адаптив в `Radio.vue` / `PersistentRadioWidget.vue`.
Для desktop-скриншота радио желательно показывать одну из тем (`Прозрачнее` или `Контрастнее`) и видимые карусели подборок/избранного.
Для главной и длинных разделов желательно, чтобы хотя бы один скриншот показывал новый глобальный action возврата в начало.
